# Contributing to The God in the Machine

Thank you for your interest in contributing. This is an opinionated architectural framework — it takes positions deliberately. Contributions that strengthen, challenge, or extend the arguments are all welcome.

## How to Contribute

### Factual Corrections

If you've identified an inaccuracy — a misattributed quote, an incorrect statistic, a mischaracterized source — please open an issue using the **Factual Correction** template. Include the specific claim, the section it appears in, and a source for the correction.

### Architectural Feedback

If you disagree with an architectural position, have implementation experience that supports or contradicts a principle, or see a gap in the framework, open an issue using the **Architectural Feedback** template.

This is especially valuable if you've built systems that align with or diverge from the principles described here. Real-world evidence is the strongest form of feedback.

### General Discussion

For broader questions, observations, or topics that don't fit the above — use the **General Discussion** template or start a Discussion thread.

### Pull Requests

Pull requests are welcome for:

- Typos, grammar, and formatting fixes
- Factual corrections (with sources)
- Improvements to cross-references between sections
- Additional references or citations

For substantive changes to the arguments or principles, please open an issue first to discuss the proposed change.

## Style Guidelines

- The paper uses a specific, deliberate voice. Please preserve the tone when suggesting edits.
- Terminology is canonical — see the Key Terminology table in the README. Do not introduce alternative terms for established concepts (e.g., don't substitute "Theomorphic" for "Theotropic").
- Claims should be attributable. If adding a factual claim, include a source.

## Code of Conduct

Be constructive. Disagree with arguments, not with people. This is an architectural discussion, not a debate stage.
